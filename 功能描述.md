# QSLRecoder — QSL 通联记录管理软件 需求描述

## 1. 项目概述

### 1.1 项目背景

QSL 卡片是业余无线电（Ham Radio）爱好者之间确认通联（QSO）的传统方式。随着通联量的增长，手动管理 QSL 卡片记录变得低效且容易出错。本项目旨在开发一款 **QSL 通联记录管理软件**，帮助爱好者高效地记录、追踪和管理通联及 QSL 卡片收发状态。

### 1.2 项目目标

- 提供便捷的通联日志记录与管理功能
- 追踪每张 QSL 卡片的发送与接收状态
- 支持业余无线电通用数据格式（如 ADIF）的导入/导出
- 提供通联统计与数据分析功能
- 界面简洁直观，操作高效

### 1.3 目标用户

- 业余无线电（Ham Radio）持证操作员
- 业余无线电俱乐部 / 集体台
- QSL 卡片收藏爱好者

## 2. 功能需求

### 2.1 通联日志管理（QSO Logging）主要功能

| 功能项 | 描述 |
|--------|------|
| 新增通联记录 | 记录一次通联的完整信息，包括：呼号、日期时间、频率/频段、模式（SSB/CW/FT8 等）、信号报告（RST）、功率、备注等 |
| 编辑通联记录 | 修改已有通联记录的所有字段 |
| 删除通联记录 | 删除通联记录（支持软删除，可恢复） |
| 批量操作 | 支持批量选择、批量修改、批量删除 |
| 通联记录查重 | 新增时自动检测是否存在重复通联（相同呼号 + 时间 + 频段 + 模式）并提示 |

### 2.2 QSL 卡片管理

| 功能项 | 描述 |
|--------|------|
| QSL 状态追踪 | 为每条通联记录标记 QSL 卡片状态：未发送、已发送、已收到回执（QSL Received）、已确认（Confirmed） |
| 发送批次管理 | 将多张待发送的 QSL 卡片按目的地/ Bureau 分组，生成发送批次 |
| QSL 卡片模板 | 支持自定义 QSL 卡片模板，包含台站信息、LOGO、卡片设计等 |
| 收发记录 | 记录每张 QSL 卡片的发送日期、收到回执日期、通过方式（直邮 / Bureau / LoTW / eQSL） |
| 待处理提醒 | 自动提醒有待发送或待确认的 QSL 卡片 |

### 2.3 数据导入/导出

| 功能项 | 描述 |
|--------|------|
| ADIF 导入 | 支持导入标准 ADIF（.adi / .adif）格式文件，自动解析并创建通联记录 |
| ADIF 导出 | 将通联记录导出为标准 ADIF 格式，兼容主流日志软件（如 N1MM、JTDX、Log4OM 等） |
| CSV 导出 | 支持导出为 CSV 格式，便于在 Excel 等工具中分析 |
| 数据备份/恢复 | 支持数据库整体备份与恢复，防止数据丢失 |
| icloud上传 | 支持记录导出为文件并上传到icloud |
| icloud恢复 | 支持将icloud上记录的数据恢复到本地 |

### 2.4 搜索与筛选

| 功能项 | 描述 |
|--------|------|
| 多条件搜索 | 按呼号、日期范围、频段、模式、QSL 状态等条件组合搜索 |
| 快速过滤 | 提供常用过滤快捷方式（如"未确认的通联"、"本月通联"等） |
| 呼号联想 | 输入呼号时自动补全/联想，基于已有通联记录 |
| 排序 | 支持按任意字段排序（时间、呼号、频段等） |

### 2.5 统计与分析

| 功能项 | 描述 |
|--------|------|
| 通联统计 | 按频段、模式、国家/地区、时间等维度统计通联数量 |
| QSL 统计 | 统计 QSL 卡片发送率、回收率、确认率 |
| 奖状追踪 | 追踪常见奖状进度（如 DX Century Club / DXCC、Worked All Zones / WAZ 等），显示已通联/已确认的国家、分区数量 |
| 可视化图表 | 以柱状图、饼图、地图等形式展示通联分布 |
| 通联日历 | 以日历视图展示每日通联情况 |

### 2.6 呼号数据库

| 功能项 | 描述 |
|--------|------|
| 呼号信息查询 | 根据呼号自动查询台站所在国家/地区、CQ 分区、ITU 分区、时区等信息 |
| 在线查询 | 可选的在线呼号查询（如 QRZ.com API），获取更详细的台站信息 |

## 3. 非功能需求

### 3.1 性能

- 支持管理 **10 万条以上** 通联记录，操作流畅无明显延迟
- 搜索/筛选响应时间 **< 500ms**
- ADIF 文件导入（1 万条记录）耗时 **< 5s**

### 3.2 可靠性

- 数据持久化，支持异常退出后数据恢复
- 关键操作（删除、批量修改）支持撤销（Undo）
- 定期自动保存 / 自动备份机制

### 3.3 易用性

- 界面简洁直观，符合桌面应用操作习惯
- 支持键盘快捷键操作（适合高频使用场景）
- 支持中英文双语界面
- 提供新手引导 / 帮助文档

### 3.4 兼容性

- 支持ISO系统

### 3.5 可扩展性

- 模块化架构设计，便于后续添加新功能
- 预留插件/扩展接口（如 LoTW/eQSL 在线同步、SDR 电台控制集成等）

## 4. 数据模型（核心实体）

### 4.1 通联记录（QSO）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| callsign | String | 对方呼号 |
| datetime_on | DateTime | 通联开始时间（UTC） |
| datetime_off | DateTime | 通联结束时间（UTC） |
| frequency | Double | 频率（MHz） |
| band | String | 频段（如 20m、40m） |
| mode | String | 模式（如 SSB、CW、FT8） |
| rst_sent | String | 发送信号报告 |
| rst_rcvd | String | 接收信号报告 |
| tx_pwr | Double | 发射功率（W） |
| qsl_sent | Enum | QSL 发送状态：Y/N/R/I |
| qsl_rcvd | Enum | QSL 接收状态：Y/N/R/I |
| qsl_sent_date | Date | QSL 发送日期 |
| qsl_rcvd_date | Date | QSL 接收日期 |
| qsl_via | String | QSL 发送途径（Bureau / Direct / LoTW / eQSL） |
| country | String | 对方国家/地区 |
| cq_zone | Integer | CQ 分区 |
| itu_zone | Integer | ITU 分区 |
| grid_locator | String | 梅登海德网格定位（如 OM89） |
| name | String | 对方操作员姓名 |
| comment | String | 备注 |
| created_at | DateTime | 记录创建时间 |
| updated_at | DateTime | 记录更新时间 |

### 4.2 QSL 发送批次

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | String | 批次名称 |
| destination | String | 目的地（国家/呼号前缀） |
| via | Enum | 发送方式：Bureau / Direct |
| send_date | Date | 发送日期 |
| status | Enum | 状态：待发送 / 已发送 / 已完成 |
| qso_ids | List\<UUID\> | 包含的通联记录 ID 列表 |
| remark | String | 备注 |
